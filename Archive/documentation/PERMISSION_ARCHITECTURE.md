## Key Points

1. **Defense in Depth**: Multiple layers of protection (middleware + decorators + view logic)
2. **Automatic**: Middleware runs on every request automatically
3. **Flexible**: Combine role-based and permission-based access
4. **Secure**: Unapproved users have limited access
5. **Auditable**: All actions logged in ActivityLog
6. **User-Friendly**: Custom 403 page with clear messaging
7. **Performance**: Fast checks with early exits
8. **Maintainable**: Centralized configuration

## Flow Summary

1. Request arrives â†’ Middleware intercepts
2. Check if public/static â†’ Allow if yes
3. Check authentication â†’ Redirect if not logged in
4. Check superuser â†’ Allow if yes
5. Check approval status â†’ Limit if not approved
6. Check role requirements â†’ Deny if no match
7. Check permission requirements â†’ Deny if no match
8. Allow access â†’ View executes
9. Response created â†’ Activity logged if needed
10. Response sent to user
